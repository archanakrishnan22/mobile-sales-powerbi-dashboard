# Mobile Sales Dashboard – Power BI Project

## Business Problem
A mobile retail company wants to understand sales performance across brands, cities, and payment methods to improve sales strategy and identify high-performing markets.

## Business Questions
- Which mobile brands generate the highest revenue?
- Which cities contribute the most to total sales?
- Which payment methods are used most frequently?
- How do sales trends vary across months and days?

## Tools Used
- Microsoft Power BI
- Data Cleaning
- Data Transformation
- Data Visualization

## Dataset Overview
The dataset includes mobile phone sales transactions across India.

Dataset contains:
- Mobile brands and models
- Sales transactions
- Cities
- Payment methods
- Customer ratings
- Monthly and day-wise sales records

Key Metrics:
- Total Sales
- Total Transactions
- Total Quantity Sold
- Average Sales Value

## Dashboard Features

### KPI Metrics
- Total Sales: 769M+
- Total Transactions: 4K+
- Total Quantity Sold: 19K+
- Average Sales Value: 40K

### Analysis Sections
- Brand-wise sales performance
- City-wise sales distribution
- Monthly sales trends
- Day-wise sales patterns
- Payment method usage
- Customer rating analysis

### Interactive Filters
Users can explore the dashboard using filters for:
- Brand
- City
- Payment Method
- Day

## Key Insights
- Apple generated the highest revenue among all brands.
- Major cities contribute the largest share of total sales.
- UPI and card payments dominate transactions compared to cash.
- Customer ratings mostly range between 4 and 5, indicating strong satisfaction.

## Business Recommendations
- Increase marketing campaigns in high-performing cities.
- Maintain higher inventory for top-selling mobile brands.
- Promote digital payment methods with discounts.
- Monitor low-rated products to improve customer satisfaction.

## Conclusion
This project demonstrates how interactive dashboards built using Power BI can transform raw sales data into actionable business insights for better decision-making.
